The gap

Capital can't trust an AI agent.

Hand over a private key and the operator can run. Use a regulated managed account and you're back to KYC, custody, multi-day onboarding — incompatible with bots that act in milliseconds. Build your own audit pipeline and every team rebuilds the same mandate enforcement, drawdown gates, slashing-on-breach plumbing.

So capital sits out, and every "12% APY MM bot" pitch is unverifiable marketing.

Forum's primitive

A mandate-bounded vault on Arc.

One contract holds depositors' USDC. The operator gets execution rights bounded by an on-chain mandate: max budget, max drawdown, receipt freshness, expiry, perf-fee cut, slash bond. The operator receives only bounded credit while the mandate is active.

Anyone can call RiskKernelV2.enforce(vault) — one tx flips the vault to PAUSED and moves slash funds out of the bond. Same tx. Permissionless. No operator click required.

How it works

Three contracts. One credit line.

step 1

Deposit USDC

Anyone calls CovenantVault.deposit(amount). Funds sit idle until the operator pulls credit, bounded by mandate.

depositors → vault.deposit()
step 2

Operator pulls bounded credit

The bot calls pullCredit(N), capped at budgetUsdc. Trades on Polymarket V2 with the builder code. Returns capital + publishes a signed receipt each cycle.

operator → pullCredit() → trade → return
step 3

Anyone enforces the mandate

Drawdown over limit? Receipt stale? Budget breach? Any third party calls RiskKernelV2.enforce(vault) → vault flips to PAUSED and the bond is slashed in one tx.

anyone → kernel.enforce() → pause + slash

Risk Markets · price agent failure on-chain

Stake YES/NO on whether a SlashBond will see a slash before expiry.

Oracle-free settlement: at expiry, anyone calls SlashMarket.settle(id) which reads SlashBond.totalSlashed() and compares to the snapshot taken when the market was created. Winners get stake back + pro-rata share of losers' pool. No protocol fee in v0.

Recomputable receipts

Anyone can verify the receipt hash in 60 seconds.

Browse live receipts →

Every TrackRecordV2 record commits to an evidenceUri + evidenceHash. The URI serves canonical JSON: order-book snapshots, fills, inventory, PnL inputs, strategy config hash, decision-trace hash.

Download the JSON, keccak256 it, compare to the on-chain hash. If they match, the operator's claim is signer-attributable, sequence-protected, replay-rejected, and time-monotonic — all enforced in the contract.

Reference verifier: keeper/src/receipt.ts and keeper/scripts/verify-receipt.mjs.

  • ▸ Strict sequence — seq = lastSeq + 1
